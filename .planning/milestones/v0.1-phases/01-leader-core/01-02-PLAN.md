    ```python
    @app.route("/register", methods=["POST"])
    async def register():
        """Register a node with the leader.
        
        Per CONTEXT.md locked decisions:
        - Payload: JSON {"node_ip": "..."} with optional {"hostname": "..."}
        - Response: JSON {"status": "registered", "peers": [...]} with full peer IP list
        - Idempotent: re-registering same IP updates state without error (REGI-04)
        - Protected by asyncio.Lock (LEAD-03, RESEARCH Pitfall 5 — registration race)
        - Peer push triggered after successful registration (REGI-03)
        """
        data = await request.get_json(silent=True)
        if not data or "node_ip" not in data:
            return jsonify({"error": "Missing required field: node_ip", "status": 400}), 400

        node_ip = data["node_ip"]
        hostname = data.get("hostname", "")

        async with registry_lock:
            is_new = node_ip not in node_registry
            node_registry[node_ip] = Node(
                node_ip=node_ip,
                hostname=hostname,
                registered_at=datetime.now(timezone.utc).timestamp(),
            )
            # Build full peer IP list (excluding self)
            peers = [ip for ip in node_registry.keys() if ip != node_ip]

        logger.info(f"Node {'registered' if is_new else 're-registered'}: {node_ip} hostname={hostname}")

        # Peer push: notify all existing nodes of the updated list (per REGI-03, CONTEXT.md)
        # Fire-and-forget pattern per OpenCode's Discretion (retry/cleanup timing)
        if peers:
            asyncio.ensure_future(_push_peers_to_all(node_ip))

        return jsonify({"status": "registered", "peers": peers}), 200
    ```

    **Add GET /node-list handler** (before `@app.errorhandler`):

    **Add peer push helper function** (add before `@app.route("/livez")` or after `main()` — top-level function in module):
    ```python
    async def _push_peers_to_all(new_node_ip: str):
        """Push updated peer list to all registered nodes' /update-peers endpoint.
        
        Per CONTEXT.md: POSTs full peer IP list to each node's /update-peers on port 58080.
        This is fire-and-forget — failures are logged, not retried (OpenCode's Discretion).
        """
        async with registry_lock:
            all_nodes = dict(node_registry)

        peers = [ip for ip in all_nodes.keys()]
        async with httpx.AsyncClient(timeout=5.0) as client:
            for ip, node in all_nodes.items():
                # Don't notify the node that just registered (it already has the list in the response)
                if ip == new_node_ip:
                    continue
                try:
                    url = f"http://{ip}:58080/update-peers"
                    await client.post(url, json={"peers": peers})
                    logger.debug(f"Peer list pushed to {ip}")
                except Exception as e:
                    logger.warning(f"Failed to push peer list to {ip}: {e}")
    ```

    **Idempotency verification** (REGI-04): The handler's `async with registry_lock` block uses `node_registry[node_ip] = Node(...)` — a dict assignment that overwrites the previous entry. This is naturally idempotent: calling it twice with the same node_ip updates hostname/registered_at but does not create a duplicate entry or return an error. The `is_new` flag distinguishes first registration from re-registration for logging purposes only.

    ```python
    @app.route("/submit", methods=["POST"])
    async def submit():
        """Accept check results from nodes (per LEAD-04).
        
        Per CONTEXT.md locked decisions:
        - Validates JSON body has 'node_ip', 'checks' (array), 'timestamp' (REQUIRED)
        - Returns 400 on bad data with JSON {"error": "...", "status": 400}
        - Returns 202 Accepted with {"status": "accepted", "count": N} on success
        """
        data = await request.get_json(silent=True)
        if not data:
            return jsonify({"error": "Request body must be JSON", "status": 400}), 400

        # Validate required fields per CONTEXT.md: node_ip, checks array, timestamp
        node_ip = data.get("node_ip")
        checks = data.get("checks")
        timestamp = data.get("timestamp")
        if not node_ip or not isinstance(node_ip, str):
            return jsonify({"error": "Missing or invalid field: node_ip (string required)", "status": 400}), 400
        if not isinstance(checks, list):
            return jsonify({"error": "Missing or invalid field: checks (array required)", "status": 400}), 400
        if not isinstance(timestamp, (int, float)):
            return jsonify({"error": "Missing or invalid field: timestamp (number required)", "status": 400}), 400

        from .models import check_results as cr_store, CheckResult

        # Parse each check into CheckResult model
        accepted = 0
        for check in checks:
            if not isinstance(check, dict):
                continue
            # Each check expected: {target_ip, check_type, status, latency_ms}
            target_ip = check.get("target_ip")
            check_type = check.get("check_type")
            status = check.get("status")
            latency_ms = check.get("latency_ms", 0.0)
            if not target_ip or not check_type or not status:
                continue
            cr_store.append(CheckResult(
                source_ip=node_ip,
                target_ip=target_ip,
                check_type=check_type,
                status=status,
                latency_ms=float(latency_ms),
                timestamp=float(timestamp),
            ))
            accepted += 1

        logger.info(f"Submitted {accepted}/{len(checks)} checks from node {node_ip}")
        return jsonify({"status": "accepted", "count": accepted}), 202
    ```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Node → POST /register | External HTTP request carrying node identity. No auth (VPN trusted network). |
| Node → POST /submit | External HTTP request carrying connectivity check data from untrusted but authenticated-by-VPN nodes. |
| Leader → Node /update-peers | Outbound HTTP push to registered nodes. May fail silently (fire-and-forget). |
| External → GET /node-list | IP disclosure to any caller on the VPN. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-06 | Tampering | POST /register payload | mitigate | Validate `node_ip` is present and is a string. Reject with 400 if missing or wrong type. |
| T-01-07 | Tampering | POST /submit payload | mitigate | Validate required fields (node_ip, checks array, timestamp). Skip malformed check entries gracefully — partial acceptance per CONTEXT.md pattern. Return count of accepted checks. |
| T-01-08 | Denial of Service | POST /register | accept | No rate limiting for prototype. `asyncio.Lock` serializes handlers so registration cannot race but a flood would serialize and delay. Accept for v1. |
| T-01-09 | Denial of Service | Peer push | mitigate | Fire-and-forget with 5s HTTP timeout. Failed pushes are logged but do not block registration response. No retry loop (per OpenCode's Discretion). |
| T-01-10 | Information Disclosure | GET /node-list | accept | Full IP list returned to any caller on VPN. This is the intended mesh discovery mechanism. |
| T-01-11 | Repudiation | POST /register, POST /submit | mitigate | All registration and submission events logged at INFO level with node_ip and timestamp. |
</threat_model>
