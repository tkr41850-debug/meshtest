# Milestone v0.9: UI Consolidation — History Bars, Color & Windows

**Status:** ✅ SHIPPED 2026-06-20
**Phases:** 21-23
**Total Plans:** 3 phases

## Overview

Unified color scheme, 90-bar windows (90m/90h/90d), shared card layout template across all views.

## Phases

### Phase 21: Color Consistency (COLOR)

**Goal**: Bar and number colors are consistent across all views using a shared HSL gradient color function
**Depends on**: Nothing
**Plans**: 1 plan

**Details:**
- Extracted shared `uptimeColor()` to `views/colors.ts`
- HSL gradient: <90%→red (0°), 90-99%→amber ramp (45°→120°), ≥99.9%→green (120°)
- Removed duplicated `uptimeColor()` from `cards.ts` and `day30.ts`
- Tests at boundary values (0%, 50%, 89.9%, 90%, 99.9%, 100%)

### Phase 22: 90m/90h/90d Window Expansion (WINDOW)

**Goal**: All three time windows display 90 bars each; backend supports 90-minute retention and a new 90-hour endpoint
**Depends on**: Nothing
**Plans**: 1 plan

**Details:**
- Backend: in-memory retention 1800s→5400s, `/data?window=90h` endpoint with hourly aggregation
- Frontend: all bars 30→90, third tab (90h) in main.ts
- api.ts: fetchData90h, renamed fetchData30m→90m, 30d→90d
- types.ts: HourData, Data90hResponse added

### Phase 23: Unified Cards Layout (UNIFY)

**Goal**: All three time windows use a consistent card layout with split circle + total check count, sharing a single card template
**Depends on**: Phase 21, Phase 22
**Plans**: 1 plan

**Details:**
- Extracted shared card template to `views/card.ts` (cardHtml, splitCircle, BADGE_MAP)
- cards.ts: includes split circle + total check count
- day30.ts: per-pair card rendering (not per-day rows)
- hourly.ts: uses shared card template

## Milestone Summary

**Key Decisions:**
- Shared color function in `views/colors.ts` — single source of truth for HSL threshold coloring
- Bars AND numbers use same `uptimeColor()` function
- Card template extracted to `views/card.ts` — reused by all three time windows
- Backend `/data?window=90h` reuses 30d aggregation pattern, groups by hour instead of day
- Bar count increased 30→90 in all views; 90m=1-min bars, 90h=1-hour bars, 90d=1-day bars

**Issues Resolved:**
- Inconsistent bar/number coloring across views
- Fixed duplicate color logic in cards.ts and day30.ts
- 30-bar window too short for meaningful history — extended to 90
- Missing intermediate 90-hour view between 90m and 90d
- Visual inconsistency between 30m cards and 30d rows

**Technical Debt:** None accumulated.
