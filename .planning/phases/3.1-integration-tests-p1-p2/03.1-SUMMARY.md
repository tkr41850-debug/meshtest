**1. [Rule 3 - Blocking] Ping timeout test failed due to second proc.wait() in exception handler**
- **Found during:** test_ping_timeout_handled_gracefully
- **Issue:** mock `proc.wait()` had `side_effect=asyncio.TimeoutError()` on all calls, but node.py's exception handler calls `proc.wait()` a second time (line 69). The second call also raised TimeoutError, which propagated unhandled.
- **Fix:** Used a counter-based async function that raises TimeoutError only on the first call and returns 1 on subsequent calls.
- **Files modified:** `tests/test_node_integration.py`

**2. [Rule 3 - Blocking] Cycle test called run_check_cycle twice due to asyncio.sleep mock**

## Issues Encountered

- `asyncio.TimeoutError` in Python 3.12 is `builtins.TimeoutError` (subclass of `Exception`), so `asyncio.wait_for` catches it from the wrapped coroutine before the `except asyncio.TimeoutError` handler in node.py gets a chance. Workaround: counter-based mock that only raises on first call, letting the handler's second `proc.wait()` succeed.
- `asyncio.CancelledError` in Python 3.12 inherits from `BaseException`, not `Exception`. This means it's NOT caught by generic `except Exception` handlers — which is why our `_StopLoop(BaseException)` approach works cleanly.
