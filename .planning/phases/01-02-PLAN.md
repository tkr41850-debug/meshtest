# Plan 01-02: Registration & Submission Endpoints

## Objective

Add node registration, peer list distribution, submission endpoint, and node-list endpoint to the leader server.

## Tasks

### Task 1: Registration Endpoint

<read_first>
- mesh_status/leader.py
- mesh_status/models.py
</read_first>

<action>
In `mesh_status/leader.py`, add these routes and helpers:

1. `POST /register`:
   - Accepts JSON body matching `NodeRegistration` (`{"node_ip": "...", "hostname": "..."}`)
   - Validates `node_ip` is present and is a valid IPv4/IPv6 string
   - Acquires `_registry_lock`
   - If `node_ip` already in `_registry`: updates `last_seen` (idempotent), still triggers peer push
   - If new: adds `NodeInfo(node_ip=..., hostname=..., last_seen=time.time())` to `_registry`
   - Releases lock
   - Calls `_push_peer_list_to_all()` (see Task 2)
   - Returns `{"status": "registered", "peers": [all registered IPs]}` with 200

2. `GET /node-list`:
   - Acquires `_registry_lock`
   - Returns `{"nodes": [all_node_ips], "count": N}` with 200

3. `POST /submit`:
   - Accepts JSON body matching `SubmitPayload` (`{"node_ip": "...", "checks": [...], "timestamp": ...}`)
   - Validates required fields: `node_ip` (string), `checks` (array, non-empty), `timestamp` (number)
   - On validation failure: returns `{"error": "Invalid payload: ...", "status": 400}` with 400
   - On success: stores checks in `_results[node_ip]` (append to list, or replace)
   - Returns `{"status": "accepted", "count": len(checks)}` with 202
</action>

<acceptance_criteria>
- `leader.py` contains `async def register()` route with `@app.route("/register", methods=["POST"])`
- `leader.py` contains `async def node_list()` route with `@app.route("/node-list", methods=["GET"])`
- `leader.py` contains `async def submit_results()` route with `@app.route("/submit", methods=["POST"])`
- Registration endpoint validates `node_ip` presence and returns 400 on missing
- Re-registering same IP returns 200 with peers list (idempotent)
- `/node-list` returns JSON with `nodes` array and `count`
- `/submit` returns 202 with `"status": "accepted"`
- All three routes are async def functions
</acceptance_criteria>

### Task 2: Peer List Push Mechanism

<read_first>
- mesh_status/leader.py
- mesh_status/config.py
</read_first>

<action>
In `mesh_status/leader.py`, add the peer push mechanism:

1. Add `_peer_notification_task: asyncio.Task | None = None` module-level variable
2. Add `_peers_by_node: dict[str, list[str]]` — tracks which IPs each node has been notified about
   - Key: node_ip, Value: list of peer IPs last sent to this node
3. Add `async def _notify_node(node_ip: str, peers: list[str])`:
   - POSTs to `http://{node_ip}:{LEADER_PORT}/update-peers` with JSON `{"peers": peers}`
   - Times out after `PEER_PUSH_TIMEOUT` seconds
   - On failure (connection error, timeout, non-2xx): log warning, do NOT retry (retry on next registration)
   - Uses `httpx.AsyncClient` for the HTTP call
4. Add `async def _push_peer_list_to_all()`:
   - Acquires `_registry_lock`, gets all current node IPs
   - For each registered node: calls `_notify_node(node_ip, all_peer_ips)`
   - Uses `asyncio.gather()` to notify all nodes concurrently
   - Updates `_peers_by_node` with the sent peer list
5. Call `_push_peer_list_to_all()` at the end of the registration handler
</action>

<acceptance_criteria>
- `leader.py` contains `async def _notify_node(node_ip, peers)` that POSTs to `http://{node_ip}:58080/update-peers`
- `leader.py` contains `async def _push_peer_list_to_all()` that notifies all registered nodes
- `_notify_node` uses `httpx.AsyncClient` with timeout
- `_push_peer_list_to_all` uses `asyncio.gather()` for concurrent notifications
</acceptance_criteria>
