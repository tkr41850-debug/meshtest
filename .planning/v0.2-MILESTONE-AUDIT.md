- CR-01: `exec` on Streamlit orphaned Hypercorn → Fixed: both in background with wait loop

### DOCK-04 — Docker Config ⚠️ GAP

**Env vars verified across all three phases:**

| Env Var | Set In | Read By | Default | Status |
|---------|--------|---------|---------|--------|
| `LEADER_HOST` | Dockerfile.leader:29 | entrypoint.sh | `0.0.0.0` | ✅ |
| `LEADER_PORT` | Dockerfile.leader:30 | entrypoint.sh | `58080` | ✅ |
| `LEADER_URL` | Dockerfile.leader:31 | dashboard.py:12 | `http://localhost:58080` | ✅ |
| `DATA_DIR` | Dockerfile.leader:32 | entrypoint.sh:8 | `/app/data` | ✅ |
| `MESH_STATUS_PORT` | Dockerfile.node:27 | config.py → `LEADER_PORT` | `58080` | ✅ |
| `MESH_STATUS_INTERVAL` | Dockerfile.node:28 | config.py → `CHECK_INTERVAL` | `10` | ✅ |
| `LEADER_IP` | Dockerfile.node:29 | node.py CMD `--leader-ip` | `leader` | ✅ |
| `NODE_IP` | Dockerfile.node:30 | node.py CMD `--node-ip` | (auto-detect) | ✅ |

**Gap:** The requirement spec DOCK-04 says node env vars should be `LEADER_URL`, `NODE_IP`, `CHECK_INTERVAL`. The implementation uses `LEADER_IP` (not `LEADER_URL`) and `MESH_STATUS_INTERVAL` (not `CHECK_INTERVAL`). These names were intentionally changed during Phase 2 code review (per 02-SUMMARY.md code review fixes WR-01 and WR-02) because the app actually consumes `LEADER_IP` (the node constructs the leader URL from `LEADER_IP` + port) and `MESH_STATUS_INTERVAL` (from config.py). The requirement spec was not updated to match.

**Recommendation:** Update the requirement spec DOCK-04 to list the actual env var names used by the implementation.

### Cross-Node HTTP Health Check Issue (pre-existing, not blocks v0.2)

```
node.py:75  GET http://{target_ip}:{config.LEADER_PORT}/healthz
```

When checking another node (e.g., node2 from node1), this requests `http://node2:58080/healthz`. The node's HTTP server (started at `node.py:175`) only has `/update-peers` and `/updateConfig` — no `/healthz` route. Cross-node HTTP checks will return 404. Documented in codebase as a known limitation; not scoped to the containerization milestone.
