    ### Test Class: `TestRegistrationFlow`

    1. `test_register_node_returns_peers(client, mock_peer_notify)`:

    2. `test_register_second_node_updates_peer_lists(client, mock_peer_notify)`:
       - Register `10.0.0.1` → peers `[]`
       - Register `10.0.0.2` → peers `["10.0.0.1", "10.0.0.2"]`
       - Register `10.0.0.3` → peers `["10.0.0.1", "10.0.0.2", "10.0.0.3"]`

    3. `test_re_registration_idempotent(client, mock_peer_notify)`:
       - Register `10.0.0.1`
       - Re-register `10.0.0.1` with same IP
       - Assert 200 status, `"status": "registered"`, no error
       - GET `/node-list` → count is 1 (no duplicates)

    4. `test_registration_missing_node_ip_returns_400(client, mock_peer_notify)`:
       - POST `/register` with empty body `{}`
       - Assert 400 status with `"error"` field

    5. `test_registration_triggers_peer_push(client)`:
       - **Do NOT use `mock_peer_notify` fixture** — use a fresh `respx.mock` with route inspection
       - Create respx mock that records calls:
         ```python
         import respx
         from httpx import URL
         with respx.mock as mock:
             # Register route BEFORE making calls
             route = mock.post(url__startswith="http://").respond(200)
             # Register A then B
             resp_a = await client.post("/register", json={"node_ip": "10.0.0.1"})
             assert resp_a.status_code == 200
             resp_b = await client.post("/register", json={"node_ip": "10.0.0.2"})
             assert resp_b.status_code == 200
             # Assert that a POST was made to notify node A about node B
             assert len(route.calls) >= 1
             # Check the call URL contains 10.0.0.1:58080/update-peers
             called_urls = [str(call.request.url) for call in route.calls]
             assert any("10.0.0.1" in url and "update-peers" in url for url in called_urls)
         ```
       - This test verifies the peer push mechanism fires on registration

    6. `test_peer_push_contains_updated_peer_list(client)`:
       - Similar approach: register A, then B
       - Inspect the POST body of the peer push call to node A
