# Milestone v0.10.1: Code Review Cleanup

**Status:** ✅ SHIPPED 2026-06-22
**Phases:** 26-30
**Total Phases:** 5

## Overview

Fix all ~40 code review findings across persistence, leader API, node agent, shell scripts, CI/Docker config, frontend, test infrastructure, and register CLI. Added spec-based integration tests that exercise the leader via HTTP only.

## Phases

### Phase 26: Persistence & Leader Core

**Goal**: Fix persistence and leader API code review findings with TDD (DATA-01 through DATA-04, LEAD-01 through LEAD-04, DEAD-01 through DEAD-04)

**Depends on**: None (base fixes)

**Details:**
- `_append_results` appends new data to existing file instead of overwriting (direct append mode, discard invalid lines with warning on read)
- `_read_results` does not sort (sole caller `load_into_memory` re-sorts anyway)
- `_ensure_data_dir` removed (dead code, never called)
- Default `DATA_ROOT` relative path documented or resolved to absolute
- `/data?window=90d` response deduplicates days with data split across `_day_aggregates` and `_results`
- `_results` and `_day_aggregates` accesses synchronized with `asyncio.Lock`
- `/updateConfig` validates input types and bounds (returns 400 on invalid input)
- Uptime percentage division by zero guarded (3 sites in `leader.py`)
- `_peers_by_node` removed from `leader.py` (declared but never populated)
- `_push_peer_list_to_all` and `_push_config_to_all` consolidated (duplicate code)
- Unused `registry` parameter removed from `calculate_status`
- `Optional[str]` replaced with `str | None` syntax (Python 3.12+)

### Phase 27: Node Agent

**Goal**: Fix node agent code review findings with TDD (NODE-01 through NODE-04)
