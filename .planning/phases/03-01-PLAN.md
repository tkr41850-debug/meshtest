### Task 2: Implement Persistence Module (GREEN)

<read_first>
- tests/test_persistence.py
- mesh_status/config.py
</read_first>

<action>
Create `mesh_status/persistence.py`:

1. `_ensure_data_dir(dt: datetime) -> Path`:
   - Creates `data/YYYY/MM/` via `os.makedirs(exist_ok=True)`
   - Returns the path

2. `_append_results(dt: datetime, results: list[dict])`:
   - Opens `data/YYYY/MM/DD.json` in append mode
   - Writes each result as a single JSON line (`.jsonl` format)
   - Uses atomic write: write to `.tmp` file, `os.rename()` to final
   - Flushes and fsyncs before rename

3. `_read_results(start_date: date, end_date: date) -> list[dict]`:
   - Reads JSON Lines from all files in date range
   - Returns parsed results sorted by timestamp

4. Background flush task:
   - `async def flush_loop(app_ctx)` — runs in `@app.before_serving`
   - Sleeps 3600 seconds
   - Copies `_results` dict, calls `_flush_results(batch)` for each node
   - Clears flushed entries from memory, keeps last 10 min

5. `_flush_results(results_batch: list[dict])`:
   - Groups by timestamp date
   - Calls `_append_results` for each date group
</action>

<acceptance_criteria>
- `mesh_status/persistence.py` exists
- All tests from Task 1 pass with `python3 -m pytest tests/ -v`
- Background flush task integrated into leader startup
</acceptance_criteria>
