  <action>
    **Part A — Add aiohttp to pyproject.toml dependencies:**

    Add `"aiohttp>=3.9.0"` to the `dependencies` array in pyproject.toml, after the existing httpx entry.

    **Part B — Add async HTTP server to node.py:**

    The node currently has no HTTP server. The leader pushes peer updates to `POST /update-peers` and config changes to `POST /updateConfig` on each node's port 58080. We need to add a lightweight aiohttp server that runs as a background task alongside the check loop.

    Implementation steps:
    
    1. Add imports at top of node.py:
       ```python
       from aiohttp import web
       ```
       Place after the existing `from mesh_status import config` import.

    2. Before the `run()` function, add handler functions:
       ```python
       async def handle_update_peers(request: web.Request) -> web.Response:
           """POST /update-peers — receive peer list and config from leader."""
           data = await request.json()
           app = request.app
           if "peers" in data and isinstance(data["peers"], list):
               app["state"]["peers"] = data["peers"]
           if "check_interval" in data:
               app["state"]["interval"] = int(data["check_interval"])
           if "buffer_size" in data:
               app["state"]["buffer_size"] = int(data["buffer_size"])
           logger.info("Updated peers via push: %d peers, interval=%s, buffer=%s",
                       len(app["state"]["peers"]), app["state"]["interval"], app["state"]["buffer_size"])
           return web.json_response({"status": "ok"})


       async def handle_update_config(request: web.Request) -> web.Response:
           """POST /updateConfig — receive config update from leader."""
           data = await request.json()
           app = request.app
           if "check_interval" in data:
               app["state"]["interval"] = int(data["check_interval"])
           if "buffer_size" in data:
               app["state"]["buffer_size"] = int(data["buffer_size"])
           logger.info("Updated config via push: interval=%s, buffer=%s",
                       app["state"]["interval"], app["state"]["buffer_size"])
           return web.json_response({"status": "ok"})


       async def start_http_server(state: dict, host: str = "0.0.0.0", port: int = 0) -> web.AppRunner:
           """Start the aiohttp server. Returns runner for cleanup."""
           app = web.Application()
           app["state"] = state
           app.router.add_post("/update-peers", handle_update_peers)
           app.router.add_post("/updateConfig", handle_update_config)
           runner = web.AppRunner(app)
           await runner.setup()
           site = web.TCPSite(runner, host, port)
           await site.start()
           logger.info("Node HTTP server listening on %s:%d", host, port)
           return runner
       ```

    3. Modify `run()` to:
       a. Create a shared state dict after parsing args, before registration:
          ```python
          # Shared mutable state accessible to both HTTP server and check loop
          shared_state = {
              "peers": [],
              "interval": config.CHECK_INTERVAL,
              "buffer_size": config.BUFFER_SIZE,
          }
          ```
       b. Start the HTTP server as a background task right after creating shared state, before registration:
          ```python
          http_runner = await start_http_server(shared_state, port=config.LEADER_PORT)
          ```
          Use `config.LEADER_PORT` (58080) — the leader pushes to this port on the node's IP.
          
          Add try/finally to ensure cleanup:
          After the main loop, add `await http_runner.cleanup()` in a finally block. Since the loop runs forever, this is for clean shutdown if an exception propagates.
          
          Actually since the existing code has no cleanup (runs forever), wrap the check loop (everything after registration) in try/except/finally and clean up the runner in finally.

       c. Use `shared_state` in the check loop instead of local variables:
          - Replace `peers = resp.json().get("nodes", [])` with updating shared_state:
            ```python
            async with httpx.AsyncClient(timeout=10.0) as client:
                resp = await client.get(f"http://{leader_ip}:{port}/node-list")
                if resp.is_success:
                    shared_state["peers"] = resp.json().get("nodes", [])
            ```
          - Replace `interval = config.CHECK_INTERVAL` with reading from shared_state:
            ```python
            interval = shared_state["interval"]
            ```
            (interval is used at the bottom of the loop for `await asyncio.sleep(interval)`)
          - Replace `buffer_size = config.BUFFER_SIZE` with reading from shared_state:
            ```python
            buffer_size = shared_state["buffer_size"]
            ```
            (used to create the deque)
          - Use `shared_state["peers"]` instead of local `peers` in the `run_check_cycle` call.

       d. Use `shared_state["interval"]` in the `asyncio.sleep(interval)` call at the bottom of the loop.

       e. The buffer_size value from shared_state is used at initialization:
          ```python
          result_buffer: deque[list[dict]] = deque(maxlen=buffer_size)
          ```
          This is set once at startup — OK for a dynamic buffer_size since maxlen can't be changed after creation. If buffer_size changes via config push, the current buffer keeps its original maxlen, which is acceptable.

    4. Do NOT change the existing `check_node`, `run_check_cycle`, `submit_results` functions — they work correctly as-is.
    5. Do NOT import anything that would cause circular imports (node.py is standalone, imports from mesh_status.config only).
    6. Do NOT add any sync HTTP server or threading — everything stays in asyncio.

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Leader → Node | Peer push / config push over HTTP to node:58080 (mesh-internal) |
| Node → Leader | Registration, submission, health checks over HTTP to leader:58080 |
| Node → Node | HTTP health checks to /healthz on peer's port 58080 |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-4.1-01 | Spoofing | node POST /update-peers | accept | Leader push is unauthenticated over mesh-internal network. Accept — consistent with existing leader endpoints (no auth on /register, /submit either). |
| T-4.1-02 | Spoofing | node POST /updateConfig | accept | Same as T-4.1-01 — no auth on mesh-internal push endpoints. Accept as design choice. |
| T-4.1-03 | Tampering | node HTTP server | mitigate | aiohttp parses HTTP requests; only two fixed routes registered — no path injection possible. JSON body validated with isinstance checks. |
| T-4.1-04 | Information Disclosure | leader GET /healthz | accept | Same as /livez — returns {"status": "alive"} only. No sensitive data. |
| T-4.1-05 | DoS | node HTTP server | accept | No rate limiting on internal mesh network. Accept — mesh nodes are trusted peers; service disruption would require network access. |
