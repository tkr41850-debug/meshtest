# Requirements: v0.2 Containerize mesh-status

## Owner

mesh-status

## DOCK-01 — Leader+Dashboard Dockerfile

**Description:** A Dockerfile (`Dockerfile.leader`) based on `python:3.12-slim` that installs dependencies and runs both the Quart API server (port 58080) and Streamlit dashboard (port 58581) as subprocesses.

**Acceptance criteria:**
- Base image: `python:3.12-slim`
- UV is installed and used for dependency management
- Exposes ports 58080 and 58581
- Entrypoint launches both Hypercorn (Quart) and Streamlit
- `docker build -t mesh-leader .` succeeds

---

## DOCK-02 — Node Agent Dockerfile

**Description:** A Dockerfile (`Dockerfile.node`) based on `python:3.12-slim` that installs `iputils-ping` and runs the standalone node agent.

**Acceptance criteria:**
- Base image: `python:3.12-slim`
- `iputils-ping` installed for the system `ping` binary
- UV is installed and used for dependency management
- Entrypoint runs the node agent (`python node.py`)
- Exposes node HTTP server port
- `docker build -t mesh-node .` succeeds

---

## DOCK-03 — Docker Compose

**Description:** A `docker-compose.yml` file that orchestrates the leader+dashboard and node agent containers for local dev/test.

**Acceptance criteria:**
- `docker compose up` starts both services
- Leader is accessible on port 58080
- Dashboard is accessible on port 58581
- Node connects to leader automatically
- Data directory bind-mounted for persistence
- Environment variables configurable via compose file

---

## DOCK-04 — Docker Config

**Description:** Docker-specific configuration including environment variables, port mappings, and volume mounts.

**Acceptance criteria:**
- Leader env vars: `LEADER_HOST`, `LEADER_PORT` (default 58080)
- Node env vars: `LEADER_URL`, `NODE_IP`, `CHECK_INTERVAL`
- Dashboard env var: `LEADER_URL`
- Data volume: `/app/data` bind-mounted for persistence
- All env vars have sensible defaults
