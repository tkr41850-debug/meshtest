<decisions>
## Implementation Decisions

### Testing Framework
- Use `pytest` with `pytest-asyncio` (already established)
- Use `respx` for mocking `httpx.AsyncClient` HTTP calls (leader-to-node peer push, node-to-leader submission)
- Use `unittest.mock.patch` for mocking `asyncio.create_subprocess_exec` (ping subprocess)
- Test files organized by integration area, not by source module

### Leader Integration Tests
- **D-01**: Use Quart test client for leader — tests exercise real Quart routing, request parsing, response serialization, CORS
- **D-02**: Mock outgoing httpx calls (peer push) with `respx` — verify leader correctly POSTs `update-peers` to all registered nodes on registration
- **D-03**: Test registration flow end-to-end: register node → verify response structure → verify peer push calls → verify node-list updated
- **D-04**: Test re-registration idempotency: same node_ip → still success, triggers peer push
- **D-05**: Test submission endpoint with valid and invalid payloads
- **D-06**: Test config push endpoint: POST /updateConfig → config values updated + push triggered
- **D-07**: Test data flow integration: submit results → query data endpoint → verify correct structure

### Node Integration Tests
- **D-08**: Mock `asyncio.create_subprocess_exec` to simulate ping success and failure — verify `check_node()` parses output correctly
- **D-09**: Use `respx` to mock leader HTTP responses (registration, /node-list, /submit) — verify node orchestration runs correct sequence
- **D-10**: Test `run_check_cycle()` with mocked `check_node()` — verify semaphore limiting and result collection
- **D-11**: Test `submit_results()` success and failure paths — verify return values
- **D-12**: Test buffer/retry logic: failed submission → buffer accumulates → successful submission → buffer cleared
- **D-13**: Test buffer respects maxlen capacity (deque eviction)

### Test Infrastructure
- **D-14**: Create `tests/conftest.py` with shared fixtures: `client` (Quart test client), `mock_peer_notify` (respx mock for outgoing HTTP), `registered_nodes` (pre-registered test nodes)
- **D-15**: Add `respx` to dev dependencies in pyproject.toml

### OpenCode's Discretion
- Exact test method names and organization within test files
- Specific mocked ping output strings (stdout, stderr) used in subprocess mock
- Number of scenarios per test file (within reason — all critical paths must be covered)
- Whether to use `asyncio.gather` or sequential calls for multi-node registration tests
</decisions>

<specifics>
## Specific Ideas

- All tests should be async (pytest-asyncio with auto mode is configured)
- For leader tests: use `app.test_client()` from Quart (already used in `test_data_api.py`)
- For node tests: import individual functions from `node.py` and test with mocked dependencies
- Buffer test: create sequence where submit fails N times, buffer grows, then submit succeeds, buffer empties
- Peer push test: register node while no other nodes exist (no push calls needed), then register second node (push calls to first node)
</specifics>
