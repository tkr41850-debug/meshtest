## Overview

Replace native HTML `title` tooltips with custom CSS-only tooltip containers in the unified card template and matrix view, using group-based hover for bar rows.

## Phases

### Phase 24: Custom Hover Tooltips

**Goal**: Replace native HTML `title` tooltips with custom CSS-only tooltip containers in the unified card template and matrix view, using group-based hover for bar rows
**Depends on**: Phase 23 (unified card template must exist before tooltip wrappers)
**Plans**: 1 plan

**Details:**
- Remove `title` attributes from bar `<span>` elements in `renderBars()`
- Add tooltip container wrappers around bar rows in `card.ts` — CSS-only hover with positioned tooltip div
- Tooltip CSS (`style.css`) — dark bg, white text, small font, rounded corners, arrow indicator
- Replace `title` on matrix column headers in `matrix.ts` with the same pattern
- Update all test assertions to check tooltip div content instead of native `title`

## Milestone Summary

**Key Decisions:**
- CSS-only approach — no JS event listeners or mouse-tracking needed
- Group-based hover for bars — one tooltip per bar row (ICMP/HTTP), not 90 individual tooltips
- Tooltip content: protocol label (ICMP/HTTP)
- Tooltip classes in `style.css` — reusable across bars and matrix views
- Arrow indicator via CSS border trick (pseudo-element)

**Issues Resolved:**
- Overwhelming 90 individual native tooltips when hovering over bar rows
- Native `title` tooltips are unstyled and browser-inconsistent
- No way to style native `title` tooltips

**Technical Debt:** None accumulated.
