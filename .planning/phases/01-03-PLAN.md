# Plan 01-03: register.py CLI Script

## Objective

Create the standalone `register.py` script that registers a node with the leader, supporting both argv and stdin input modes.

## Tasks

### Task 1: Registration Script

<read_first>
- .planning/phases/01-CONTEXT.md
</read_first>

<action>
Create `register.py` at project root:

1. Uses `argparse` for CLI:
   - `--node-ip` / `-n`: Node's own IP address (str)
   - `--leader-ip` / `-l`: Leader's IP address (str)
   - `--port` / `-p`: Leader port (int, default 58080)

2. Logic:
   - If both `--node-ip` and `--leader-ip` are provided via argv: use them directly
   - If either or both are missing: prompt via `input()`:
     - `"Enter node IP: "` for node IP
     - `"Enter leader IP: "` for leader IP
   - Constructs URL: `http://{leader_ip}:{port}/register`
   - POSTs JSON `{"node_ip": node_ip}` to the URL
   - On success (2xx): prints `"Registered successfully. Peers: {peers}"`
   - On failure: prints error message and exits with code 1
   - Uses `urllib.request` from stdlib (no additional dependencies for this script)

3. Script must be directly runnable: `python register.py --node-ip 10.0.0.1 --leader-ip 10.0.0.2`
   and also work with stdin: `python register.py` followed by interactive input
</action>

<acceptance_criteria>
- `register.py` exists at project root and has `#!/usr/bin/env python3` shebang
- Running `python register.py --node-ip 10.0.0.1 --leader-ip 10.0.0.2` attempts POST to `http://10.0.0.2:58080/register`
- Running `python register.py` without args prompts for node IP and leader IP via `input()`
- Script uses `urllib.request` (no external deps)
- Script exits with code 1 on HTTP error, code 0 on success
</acceptance_criteria>
