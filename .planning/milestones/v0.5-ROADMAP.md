# v0.5 Roadmap Archive — Frontend Migration

**Completed:** 2026-06-19

## Goal

Replace Streamlit dashboard with a Vite + TypeScript + Tailwind CSS frontend served by the leader on port 58080.

## Phases

### Phase 8: Frontend Scaffold + Build Pipeline

- Vite + TS + Tailwind + Vitest project initialized in `frontend/`
- Docker multi-stage build (Node 22-alpine → Python 3.12-slim)
- Quart static serving of `dist/` at `/` and `/assets/*`
- `entrypoint.sh` starts only Hypercorn
- compose.yml port 58581 removed
- GHA workflow builds frontend transparently

### Phase 9: Dashboard Views

- Connectivity matrix: N×N table with colored status dots and short labels with hover
- Detail cards: per-target with status badge, latencies, inline uptime %, matching v0.4 design
- 30-day view: nested expanders with daily aggregated uptime, diagonal split circles per protocol, scrollTo anchors for Phase 11
- Auto-refresh every 10s via `setInterval`, tab switching between views

### Phase 10: Streamlit Cleanup

- `dashboard.py` and `test_dashboard.py` deleted
- `streamlit` and `requests` removed from dependencies
- Vitest coverage expanded to 17 tests
- 51 Python tests remain

### Phase 11: Uptime History Visualization

- 30 horizontal bars per pair (one per day), split diagonally for ping/http
- Inline SVG sparkline showing uptime trend
- ScrollTo integration from Phase 9 30-day cells
- History tab with nested expander layout
- 22 total Vitest tests
