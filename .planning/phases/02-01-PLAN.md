  <name>task 2: Ping + HTTP health check functions with independent timeout handling and cancellation-safe subprocess</name>
  <files>node.py</files>
  <action>
Add these async functions to `node.py`:

### `async def ping_check(target_ip: str) -> tuple[bool, float | None]`
Per D-03, D-04 decisions:
- Uses `asyncio.create_subprocess_exec("ping", "-c", "1", "-W", "5", target_ip, stdout=asyncio.subprocess.PIPE, stderr=asyncio.subprocess.PIPE)` — NEVER use `subprocess.run()` or `subprocess.Popen()` (per NODE-09, research pitfall #2)
- Cancellation-safe pattern per NODE-10 (research pitfall #4):
  ```python
  try:
      await asyncio.wait_for(process.wait(), timeout=5.5)  # slightly more than -W 5
  except asyncio.TimeoutError:
      process.kill()
      await process.wait()  # prevent zombie — must await after kill
      return (False, None)
  ```
- After successful wait, read stdout: `stdout = (await process.stdout.read()).decode("utf-8", errors="replace")`
- Parse with regex: `re.search(r'time=(\d+\.?\d*)\s*ms', stdout)`
- If match found: return `(True, float(match.group(1)))`
- If process returned non-zero (ping failed): return `(False, None)`
- If regex doesn't match but process returned 0: return `(True, None)` — ping succeeded but latency unparseable
- Per CONTEXT.md D-04: after `process.kill()`, must `await process.wait()` to clean up zombie — this is the cancellation-safe pattern

### `async def http_check(target_ip: str, port: int) -> tuple[bool, float | None, int | None]`
Per D-05, D-06 decisions:
- Uses `httpx.AsyncClient(timeout=httpx.Timeout(5.0))` as context manager
- Record start time: `t0 = time.monotonic()`
- `resp = await client.get(f"http://{target_ip}:{port}/healthz")`
- Elapsed: `elapsed = time.monotonic() - t0`
- If `resp.status_code` in 200-299 range: return `(True, round(elapsed * 1000, 1), resp.status_code)`
- If non-2xx: return `(False, round(elapsed * 1000, 1), resp.status_code)`
- On any exception (`httpx.TimeoutException`, `httpx.ConnectError`, etc.): return `(False, None, None)`
- Per D-02 (independent timeout): HTTP timeout is independent from ping timeout. If HTTP times out, it returns `(False, None, None)` — does NOT affect the ping check for the same peer.

### Regex extraction
Per D-07 (OpenCode's discretion on exact regex):
- Use `re.compile(r'time=(\d+\.?\d*)\s*ms')` for clean match
- Compile the regex at module level (not inside the function) for performance

### Key design requirements
- Both functions are **pure async** — no blocking calls, no sync I/O
- Both functions handle their own exceptions and NEVER raise — always return a tuple
- Both functions are fully independent: calling `ping_check` does not affect `http_check` and vice versa (per NODE-04)

### `async def run_checks(node_ip: str, leader_ip: str, port: int, interval: int, semaphore: asyncio.Semaphore)`
The main check scheduler (per CONTEXT.md D-02):
```python
async def run_checks(node_ip, leader_ip, port, interval, semaphore):
    logger.info("Starting check cycle loop (interval=%ds)", interval)
    while True:
        cycle_start = time.monotonic()
        try:
            # 1. Fetch peer list
            peers = await fetch_peer_list(leader_ip, port)
            peers = [p for p in peers if p != node_ip]  # exclude self

            if not peers:
                logger.debug("No peers to check (only node in mesh)")
            else:
                # 2. Run checks with semaphore limiting (per NODE-08)
                # Use semaphore to wrap the gather so we don't start too many concurrent check_peer calls
                # Actually: semaphore limits how many check_peer coroutines run concurrently
                async def checked(target):
                    async with semaphore:
                        return await check_peer(node_ip, target, port)

                tasks = [checked(p) for p in peers]
                results = await asyncio.gather(*tasks, return_exceptions=True)

                # Filter out exceptions (shouldn't happen but be safe)
                valid_results = [r for r in results if isinstance(r, CheckResult)]

                # 3. Submit results
                if valid_results:
                    success = await submit_results(node_ip, leader_ip, port, valid_results)
                    if success:
                        logger.info("Cycle complete: %d/%d checks submitted", len(valid_results), len(peers))
                    else:
                        logger.warning("Submission failed — will retry next cycle")

        except asyncio.CancelledError:
            raise
        except Exception as e:
            logger.error("Check cycle error: %s", e, exc_info=True)

        # 4. Sleep
        elapsed = time.monotonic() - cycle_start
        sleep_time = max(0, interval - elapsed)
        logger.debug("Cycle took %.1fs, sleeping %.1fs", elapsed, sleep_time)
        await asyncio.sleep(sleep_time)
```

### Integration wiring
In `main()`, after registration:
```python

```

### Key design requirements
- The semaphore is used to limit `check_peer` calls, NOT individual ping/http calls within one peer-check (per NODE-08)
- `asyncio.gather` with `return_exceptions=True` ensures one peer failure doesn't block others
- The cycle timing uses `time.monotonic()` (not `time.time()`) for accurate sleep calculation
- `asyncio.CancelledError` is re-raised, not caught by the broad `except Exception`
- Self is excluded from the peer list to avoid checking own connectivity
