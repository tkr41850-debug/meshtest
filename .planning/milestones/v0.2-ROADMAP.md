# Roadmap: v0.2 Containerize mesh-status

## Goal

Dockerize the leader+dashboard and node components for easy deployment across multi-VM VPN WAN.

## Success Criteria

- Single `docker compose up` starts a working mesh-status deployment locally
- Leader+dashboard accessible at port 58080 (API) and 58581 (dashboard)
- Node container connects to leader and runs check cycles
- Data persists across container restarts (bind-mounted data volume)

## Phases

### Phase 1: Dockerfile — Leader+Dashboard

**Goal:** Create a Dockerfile that runs both Quart (58080) and Streamlit (58581) in a single container.

**Tasks:**
1. Create `Dockerfile.leader` based on `python:3.12-slim`
2. Install UV in the image, then `uv sync` to install dependencies
3. Expose ports 58080 and 58581
4. Entrypoint: launch both Hypercorn (Quart) and Streamlit as subprocesses
5. Test: `docker build -t mesh-leader . && docker run -p 58080:58080 -p 58581:58581 mesh-leader`

**Requirements covered:** DOCK-01, DOCK-04

---

### Phase 2: Dockerfile — Node Agent

**Goal:** Create a Dockerfile for the standalone node agent.

**Tasks:**
1. Create `Dockerfile.node` based on `python:3.12-slim`
2. Install `iputils-ping` for the system `ping` binary
3. Install UV, then `uv sync`
4. Expose port for node HTTP server
5. Entrypoint: run the node agent via `uv run python node.py`
6. Test: `docker build -t mesh-node . && docker run mesh-node`

**Requirements covered:** DOCK-02, DOCK-04

---

### Phase 3: Docker Compose + Deployment Docs

**Goal:** Create docker-compose.yml for local dev/test and update README with deployment instructions.

**Tasks:**
1. Create `docker-compose.yml` with leader service and node service
2. Configure environment variables, ports, volumes (data directory)
3. Write README section: Docker-based deployment
4. Test: `docker compose up` and verify /livez, /readyz, /healthz

**Requirements covered:** DOCK-03, DOCK-04, DOCK-05
