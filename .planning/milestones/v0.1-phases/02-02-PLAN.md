# Plan 02-02: Buffer/Retry + Config Push

## Objective

Add FIFO buffer with configurable capacity to node for failed submissions, and leader-side `POST /updateConfig` endpoint that pushes configuration changes to all nodes.

## Tasks

### Task 1: Node Buffer and Retry

<read_first>
- node.py (from 02-01)
</read_first>

<action>
Modify `node.py` to add buffering:

1. Add `_result_buffer: collections.deque` instance variable with `maxlen=20000`
   - Configurable via `MESH_STATUS_BUFFER_SIZE` env var or leader config push
2. After `submit_results()` call in the main loop:
   - On success: if buffer has data, try to submit buffered data as well
   - On failure: append current cycle's results + existing buffer to buffer
   - Log buffer size: "Buffer: {N} cycles queued"
3. On submit, combine buffer + current results into one payload:
   - Buffer entries are per-cycle payloads
   - On successful submit of buffered data: clear buffer
</action>

<acceptance_criteria>
- node.py imports `collections.deque`
- Buffer has maxlen=20000 (configurable)
- Failed submissions append to buffer
- Successful submissions clear buffer
- Buffer contents are retried on next cycle
</acceptance_criteria>

### Task 2: Leader Config Push Endpoint

<read_first>
- mesh_status/leader.py
- mesh_status/config.py
</read_first>

<action>
Modify `mesh_status/leader.py`:

1. Add `POST /updateConfig` route:
   - Accepts JSON `{"check_interval": 5}` or `{"buffer_size": 10000}`
   - Updates `config.CHECK_INTERVAL` and `config.BUFFER_SIZE` in-memory
   - Calls `_push_config_to_all()` to propagate to all nodes
   - Returns `{"status": "config_updated", "config": {...current config...}}` with 200

2. Add `async def _push_config_to_all()`:
   - Similar to `_push_peer_list_to_all()`
   - POSTs current config to each node's `/updateConfig` endpoint on port 58080
   - Sends: `{"check_interval": CHECK_INTERVAL, "buffer_size": BUFFER_SIZE, "peers": [...]}`

3. Modify `_notify_node()` to also send config alongside peer list
   - Or create separate `_notify_node_config()` function

Modify `mesh_status/config.py`:

4. Add `BUFFER_SIZE = int(os.environ.get("MESH_STATUS_BUFFER_SIZE", "20000"))`

Modify `node.py`:

5. Add lightweight HTTP server or endpoint handler for `/updateConfig`:
   - Since node runs on same port, use `asyncio.start_server` or a simple HTTP handler
   - Accept POST to `/updateConfig`, parse JSON, update local config (check_interval, buffer_size)
</action>

<acceptance_criteria>
- `leader.py` has `POST /updateConfig` route
- `leader.py` has `_push_config_to_all()` function
- `config.py` has `BUFFER_SIZE = 20000`
- Leader pushes config to all nodes when /updateConfig is called
- Node accepts POST to /updateConfig and updates interval/buffer config
</acceptance_criteria>

## Verification

### must_haves
- POST /updateConfig to leader returns 200 and updates config
- Leader pushes config to all registered nodes
- Node buffers failed submissions up to configured capacity
- Node retries buffered data on next successful submission
- Config changes propagate to nodes' check interval and buffer size
