# Plan 02-01: Node Agent Core

## Objective

Create the standalone `node.py` script with async loop: fetch peer list, run concurrent ICMP ping + HTTP /healthz checks, submit results.

## Tasks

### Task 1: Node Script Scaffold and Registration

<read_first>
- .planning/phases/02-CONTEXT.md
- mesh_status/models.py
- mesh_status/config.py
</read_first>

<action>
Create `node.py` at project root:

1. Shebang: `#!/usr/bin/env python3`
2. Imports: `asyncio`, `logging`, `sys`, `time`, `json`, `re`, `argparse`, `httpx`, `mesh_status.models`, `mesh_status.config`
3. Parse args with argparse:
   - `--leader-ip` / `-l`: Leader IP (required from argv or stdin)
   - `--port` / `-p`: Leader port (default 58080)
4. Configure logging: same pattern as leader (INFO to stderr)
5. On startup:
   - Get leader IP from args or stdin prompt
   - Get own IP via `--node-ip` arg or auto-detect (socket.gethostbyname(socket.gethostname()))
   - Register with leader via POST /register with `{"node_ip": own_ip}`
   - Store peer list from registration response
</action>

<acceptance_criteria>
- `node.py` exists with shebang
- Running `python node.py --leader-ip 10.0.0.2` connects to leader
- Registration POST is sent to `http://{leader_ip}:{port}/register`
- Peer list is stored from registration response
</acceptance_criteria>

### Task 2: Check Execution Engine

<read_first>
- node.py (from Task 1)
</read_first>

<action>
Add check execution to `node.py`:

1. `async def check_node(target_ip: str, timeout: float = 5.0) -> dict`:
   - Runs ICMP ping AND HTTP GET /healthz against target_ip
   - Ping: `asyncio.create_subprocess_exec("ping", "-c", "1", "-W", str(timeout), target_ip, stdout=asyncio.subprocess.PIPE, stderr=asyncio.subprocess.PIPE)`
   - Cancellation-safe timeout pattern: `await asyncio.wait_for(process.wait(), timeout=timeout+0.5)`
   - If timeout: `process.kill()` then `await process.wait()`
   - Parse ping stdout with regex `time=(\d+\.?\d*)\s*ms` for latency
   - HTTP: `httpx.AsyncClient(timeout=timeout).get(f"http://{target_ip}:58080/healthz")`
   - Return dict: `{"target_ip": target_ip, "ping_ok": bool, "ping_latency_ms": float|None, "http_ok": bool, "http_status": int|None, "http_latency_ms": float|None, "timestamp": time.time()}`

2. `async def run_check_cycle(semaphore: asyncio.Semaphore, peers: list[str]) -> list[dict]`:
   - Uses semaphore to limit concurrency
   - Gathers all peer checks with `asyncio.gather(*[check_node(ip) for ip in peers])`
   - Returns list of check results

3. `async def submit_results(results: list[dict], leader_ip: str, port: int)`:
   - POSTs to `http://{leader_ip}:{port}/submit` with JSON `{"node_ip": own_ip, "checks": results, "timestamp": time.time()}`
   - Returns True on success (2xx), False on failure
</action>

<acceptance_criteria>
- `check_node()` uses `asyncio.create_subprocess_exec` for ping (not subprocess.run)
- Cancellation pattern uses `process.wait()` not `communicate()`
- Ping regex captures `time=XX ms` from stdout
- HTTP health check uses `httpx.AsyncClient`
- `run_check_cycle()` uses semaphore and asyncio.gather
- Both ping and HTTP have independent timeout handling
</acceptance_criteria>

### Task 3: Main Loop

<read_first>
- node.py (from Tasks 1 & 2)
</read_first>

<action>
Add the main loop to `node.py`:

1. `async def run()`:
   - Start: register with leader, get initial peer list and interval
   - Create semaphore: `asyncio.Semaphore(10)` (max 10 concurrent checks)
   - Loop:
     a. Fetch updated peer list from leader GET /node-list every cycle
     b. Run `run_check_cycle(semaphore, peers)`
     c. Submit results with `submit_results()`
     d. On submission failure: buffer results (stored in instance variable)
     e. If buffer has data: include buffered data in next submission
     f. Sleep for `interval` seconds (default from config)

2. Add `asyncio.run(run())` at script bottom in `if __name__ == "__main__":`
</action>

<acceptance_criteria>
- Node loops: fetch peers → run checks → submit → sleep
- Fetches peer list via GET /node-list at start of each cycle
- Uses semaphore to limit concurrent checks
- Runs as `asyncio.run(run())`
</acceptance_criteria>
